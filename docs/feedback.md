需求背景
haigoo的目标是帮助互联网人快速找到靠谱的远程工作机会
互联网从业者在寻找远程工作时普遍面临以下问题：
- 岗位不可信：中介诈骗，虚假机会，职位信息不透明
- 信息碎片化：真实岗位散落在多个渠道，收集成本高
- 缺乏方向感：无法快速发现真正适合自身经验与薪资预期的机会
这些问题导致用户陷入“浪费时间—降低信任—降低动力”的负循环。
因此，职位列表页（Job Discovery Page）核心价值是通过 高密度的信息呈现方式，让用户能在3s内：
- 真实: 这不是爬虫乱抓的垃圾，是经过 Haigoo 验证的
- 可及: 这是中国开发者真正能投递的（时区/语言匹配）
- 高价值：信息有用，是值得我付费获取投递机会渠道

用户分类及User Journey
- A. 主动求职者（主动搜索）- 希望快速找到可以投递的岗位
  - 首页 → 远程岗位 → 远程岗位页筛选/浏览岗位 → 点击岗位详情 → 决策是否付费/投递
  - 首页 → 搜索 → 远程岗位页筛选/浏览岗位 → 点击岗位详情 → 决策是否付费/投递
- B. 被动求职者（浏览机会）- 来到首页后可以快速浏览内容
  - 首页 → 下拉精选岗位 → 点击岗位详情 → 决策付费/是否投递

1. 整体布局架构
- 页面形态：采用 居中单列流式布局。去除左右宽大的侧边栏，将屏幕核心 C 位完全留给产品介绍+搜索框+岗位列表。
- 视觉风格：极简、留白、商务科技感。卡片化设计，利用微阴影区分层级。
2. Banner / 顶部通告
- 维持现状（如有活动或公告可在此展示，否则隐藏以节省空间）。
3. 搜索区 (Hero Section)
- 功能定位：视觉的第一落脚点，负责建立信任并提供入口。
- 布局逻辑：
  - 紧凑高度：严格控制高度，确保在首屏 (Fold) 能够露出 至少 1-2 个 下方的岗位卡片，让用户一眼看到“干货”。
  - 价值主张：大标题居中 —— “为中国人才打开全球远程工作入口”。
  - 超级搜索框：占据视觉中心，尺寸宽大，Placeholder 提示可搜索“公司/职位/技能”。
  - 信任背书：在搜索框下方展示三个关键指标，“9,342 精选岗位数” “617 累计Offer数” “112 日均新增岗位数” ，用数字建立平台靠谱的第一印象。
4. 精选岗位 (Feed 流)
- 推荐策略：信任优先，兼顾吸引力。
  - 进行岗位职能打散，并穿插高薪岗位- 每10个坑位必须每个一级职能（产品/研发/数据 etc.）必须都有至少一个，且其中1-4-8坑位都是对应职能候选池里的高薪岗位倒叙
- UI 设计：完全复用【远程工作】页的瀑布流设计，但是不展示筛选框
