# Haigoo 公开 Beta · 90 天 GTM 执行方案

## 1. 当前判断
- 产品已进入“可用、可验证、可开始小范围公开获取用户”的阶段。
- 现阶段不适合大规模推广，核心原因是价值主张仍然分散，用户未必能快速理解为什么现在就要留下来并付费。
- 近期目标不是追求规模化增长，而是验证一个最小商业闭环是否成立。

## 2. 当前统一定位
- Haigoo = 帮中国候选人找到并判断“值得投、能投、适合投”的远程岗位。
- 第一阶段重点服务：
  3-8 年经验、具备基础英文协作能力、希望在 3 个月内拿到远程机会的中国知识工作者。
- 第一阶段内容和案例优先围绕营销、运营、产品这类更容易形成内容传播与案例闭环的岗位簇。

## 3. 对外产品结构
- Free：
  可信岗位浏览、基础筛选、基础申请、公开 Beta 社群、有限收藏与体验。
- Pro Subscription：
  更高质量的判断信息、执行工具、Copilot 深度结果、投递管理、个性化推荐与提醒。
- Assisted Service：
  简历精修、策略诊断、模拟面试、求职陪跑等高客单人工服务，单独交付，不与订阅混卖。

## 4. 90 天分阶段目标

### 0-30 天
- 统一首页、会员页、社群页和内容口径，全部回到“可信岗位 + 决策效率”主线。
- 创始人亲自完成 20-30 个目标用户访谈，沉淀真实语言和主要阻力。
- 搭建 10-20 人高互动种子群，优先观察高意向用户的真实留存与讨论质量。
- 每周输出 3 类内容：
  可信远程岗位精选、远程求职判断方法、真实投递/面试/offer 复盘。

### 31-60 天
- 跑通第一笔订阅或轻服务闭环。
- 主推低门槛 Pro 方案，不主推长期大额套餐。
- 沉淀 3-5 个可公开的真实案例。
- 把社群定位从“信息分发”升级为“求职进展陪伴”。

### 61-90 天
- 只放大已经证明有效的自然渠道。
- 开始做转介绍机制与案例裂变。
- 若留存与付费稳定，再准备更清晰的季度订阅放量节奏。
- 若留存与付费不稳定，继续收窄 ICP，不以加更多功能替代 PMF 验证。

## 5. 关键指标
- 新注册用户首日激活率：30%-40%
- 激活用户 4 周留存：>= 20%
- 激活用户到付费用户转化：5%-8%
- 可公开复述的有效案例：>= 5 个
- 至少 1-2 个自然渠道可持续带来高意图用户

## 6. 当前阶段不做
- 大预算广告投放
- 泛人群品牌曝光
- 重 B 端销售
- 复杂企业合作
- 纯为“显得更完整”而继续堆功能

## 7. 创始人资源分配建议
- 40%：用户访谈与成交
- 30%：内容输出与品牌表达
- 20%：产品与数据复盘
- 10%：BD / 合作试探
