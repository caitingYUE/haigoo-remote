# 职位组合：会员定制内容方案

## 目标与边界

职位组合用于交付一个会员的定制求职内容：岗位清单与职业成长路径在同一页呈现。它不是岗位库的访问门票，也不承诺录用、代投或代面试。

管理员可创建公开、Club、指定用户或仅管理员可见的组合。指定用户模式通过已注册账户的 `user_id` 授权；邮箱仅作为后台展示和审计快照，因此用户修改邮箱后不会丢失访问权。

## 管理端交互

1. 填写标题、说明、有效期、优先级和启用状态。
2. 选择可见范围；选择“指定注册用户”时，按邮箱或用户名搜索站内活跃用户并逐个加入，可重复搜索添加多人。
3. 从已发布的职业成长视频中按顺序加入路径，为每条视频填写仅组合页可见的使用说明。
4. 搜索并排序职位；组合允许只含成长内容、只含职位或两者同时存在。
5. 列表显示职位数、成长内容数与指定用户数，管理员无需进入编辑页即可判断交付是否完整。
6. 保存前校验标题、时间窗、指定用户数量与视频/职位数量；保存失败显示可操作的原因，不使用泛化提示。

## 前台交付

- 左栏：标题、副标题、服务说明、有效期和职位列表；职位列表单独滚动。
- 右栏：成长内容与“我的记录”两个 Tab；右栏独立滚动。
- 成长内容按顺序展示视频、组合专属使用说明与完成状态。
- 我的记录由当前登录用户维护，记录申请、复盘和成长笔记；每条记录保留创建时间。
- 未登录用户可看到公开组合；指定用户组合返回不可用状态，避免泄露标题、成员名单或内容。

## 权限和数据规则

| 可见范围 | 查看规则 | 交付规则 |
| --- | --- | --- |
| 公开 | 所有人 | 正常显示 |
| Club | 可查看入口；非有效 Club 显示权益引导 | 有效 Club 才可查看职位和内容 |
| 指定用户 | 登录账户 `user_id` 在允许名单内 | 仅被授权账户可查看与记录 |
| 仅管理员 | 普通前台不可见 | 用于配置/预览，不作为交付链接 |

- 组合详情、列表和进度写入都检查启用状态和开始/结束时间。
- 有效期按服务器时间判断；后台应明确展示和保存时区，避免本地电脑与数据库时钟不一致时误判为“尚未开放”或“已结束”。
- 成长视频必须来自已发布视频；下线视频不会在前台渲染。
- 职位选择器应只允许加入可在前台展示的岗位；若岗位在发布后下线，前台需显示“该岗位已下线”的替代状态，而不是静默变成空合集。
- 前台记录按 `bundle_id + user_id` 隔离；最多 100 条、单条最多 2,000 字。
- 管理端 API 必须验证管理员身份；用户搜索接口只返回选择所需的最小字段。

## 已纳入的数据模型

`job_bundles`：`career_items`（排序后的视频与说明）、`allowed_user_ids`（权限来源）、`allowed_emails`（展示/审计快照）。

`job_bundle_user_progress`：`completed_video_ids`、`growth_records`、更新时间，以 `bundle_id + user_id` 唯一。

## 后续运营与技术建议

1. 增加管理员操作审计：创建、授权变更、撤销、启停及内容发布人。
2. 为成长记录增加编辑/删除与可选的“申请 / 复盘 / 学习”类型；当前版本先保证快速记录和隐私隔离。
3. 管理端增加“以用户视角预览”和授权人数提示，降低配置后才发现不可访问的风险。
4. 若指定用户规模扩大，改为 `job_bundle_access_grants` 关系表，替代 JSON 数组，便于检索、审计和批量授权。
5. 上线前补充端到端用例：管理员保存、指定用户访问、未授权用户拒绝、到期拒绝、邮箱修改后仍可访问、视频下线后的降级展示、进度并发写入。
6. 将本地预览岗位与真实可发布岗位分开标识；当前预览数据可能不在 `jobs` 表中，直接放入组合会在前台被安全过滤。
