### Haigoo 会员方案 PRD（V1.0）

目标：
在不破坏用户信任、不制造“岗位门票感”的前提下，通过会员制将 Haigoo 的核心价值——「可信过滤 + 决策支持 + 内推执行」产品化，并为后续服务化（简历、内推、求职陪跑等）打下基础。

1. 产品定位与设计原则
1.1.1 产品一句话定位

Haigoo = 经过人工审核的远程岗位可信入口 + 会员决策过滤器

价值点在于：
- 帮用户降低 80% 无效筛选成本
- 帮用户避开高风险 / 低反馈 / 不适配岗位
- 帮用户把“投简历”变成可管理、可判断的过程

1.1.2 核心设计原则（不可破坏）
1. 不拦真实申请入口（官网 / 第三方平台）
2. 不伪造申请进度
3. 会员买的是判断力与执行效率，而不是岗位本身
4. 免费用户也能找到工作，但会更累；会员明显更轻松

3. 岗位来源与统一抽象
3.1 岗位来源类型
1. 第三方平台岗位（RSS 等方式引入）
2. 企业官网岗位（可信官网岗位爬取）
3. Haigoo 合作内推岗位（人工录入）

3.2 前端统一展示原则
- 所有岗位均通过 Haigoo 底线人工审核
- 对用户来说：
  - 都是可信岗位
  - 区别在于「信息深度」与「执行方式」

4. 功能权限矩阵（核心）
4.1 免费用户（Free）功能

目标：建立信任 + 感知价值，但产生使用摩擦

1. 岗位浏览
- 查看岗位列表（瀑布流）
- 查看完整 JD
- 基础筛选（岗位类型 / 是否远程 / 国家）
- 基础岗位标签等
- ……

2. 申请行为
- 可点击 Apply
- 跳转前展示「Haigoo 已审核说明弹层」
- 跳转后仅显示：
  - 状态：已跳转申请（无追踪）

3. 限制点（关键）
- 无企业风险评估
- 无会员点评
- 无投递记录管理
- 无内推岗位申请权限
- 收藏数量有限（如 3–5 个）


4.2 会员用户（Member）功能（核心付费点）

目标：让用户明显感受到“我是在理性投递，而不是盲投”
A. 岗位信息增强（核心付费价值）
1. 企业风险 / 友好指数（人工 + 数据）
- 中国候选人友好度（★1–5）
- 历史回复率（低 / 中 / 高）
- 平均反馈周期（区间）
- 常见风险提示

2. Haigoo 人工点评（一句话）
- 示例：
- “官网真实、岗位长期存在，但回复偏慢，适合耐心投递”

3. 隐藏字段解锁
- 时区偏好
- 英文真实要求
- 是否偏向全职 / 合同

B. 搜索与决策能力
- 高级筛选：
  - 时区
  - 英文要求强度
  - 合同类型
- 岗位收藏（不限）
- 岗位对比（2–3 个）
- 「是否适合我」快速判断模块

C. 投递与执行管理
非内推岗位（官网 / 第三方）
- 自动记录跳转投递行为
- 状态：已跳转申请（待结果）
- 用户可手动标记：
  - 已回复
  - 已面试
  - 已拒绝 / 已 Offer

内推岗位（Haigoo 核心壁垒）
- 完整 JD 可见
- 会员专属申请入口：
  1. 选择 / 上传简历
  2. 简单补充问题（可选）
  3. 提交
- 状态流转：
  - 已提交 → 已审核 → 已转递 → 已反馈

5. 关键交互流程设计
5.1 Apply 申请统一流程（重要）
场景 A：官网 / 第三方岗位
Free 用户：
- 点击 Apply
- 弹层说明：
  - Haigoo 已人工审核
  - 中国人可投
  - 非骗局
- CTA：
  - 继续前往申请
  - 解锁会员查看风险评估

Member 用户：
- 直接跳转
- 自动记录投递
- 返回后提示：
已为你记录申请，可在「我的投递」查看

场景 B：内推岗位
Free 用户：
- JD 可见（可轻度模糊）
- 点击申请 → 会员引导弹层

Member 用户：
- 3 步内完成申请
- 明确 SLA 提示（如 X 个工作日审核）

5.2 会员引导策略（非强拦）
触发点：
- 查看风险评估
- 申请内推岗位
- 收藏超限
- 查看投递管理

引导文案原则：
- 对比而非威胁
- 强调节省时间与避坑

6. 支付与会员开通方案
阶段一：无支付接口（当前阶段）
方式：私域 + 人工开通（推荐）

链路设计
1. 页面提示：
Member 功能目前采用邀请制
2. CTA：
- 申请加入 Member（跳转微信）
3. 私域流程：
- 简要介绍 Haigoo
- 2–3 个筛选问题
- 告知体验期 / 内测价
4. 后台人工赋权：
- Member = true
- 到期时间

前端展示：
- Member · 特邀用户
- 有效期至 XX

阶段二：正式支付（后续）
建议定价（参考）
- 月度会员：¥39–69
- 季度会员（主推）：¥99–149
- 7 天体验：¥19.9

支付不作为 PRD 第一阶段强依赖。

7. 核心付费点总结
1. 企业风险与友好评估（不可免费）
2. 投递管理与反馈能力
3. 内推岗位的真实执行权
4. 从“看岗位”到“做决策”的跃迁

8. MVP 实现优先级
P0（必须）
- 会员字段与权限判断
- 岗位认证标签
- Apply 前弹层
- 收藏 & 投递记录
- 内推岗位独立流程

P1（增强）
- 风险指数体系
- 用户反馈数据
- 岗位对比

9. 成功指标（北极星）
- 免费 → Member 转化率
- Member 用户周活
- 内推岗位申请完成率
- 用户主观反馈：
- “这个网站让我少走了很多弯路”
