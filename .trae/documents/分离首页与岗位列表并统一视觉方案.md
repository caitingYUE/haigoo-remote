## 目标与范围
- 目标：分离首页与岗位列表，统一视觉风格；在首页提供“我在国内/我在海外”两大入口与订阅（邮件+飞书）；列表页采用浅色背景+岗位卡片，无 Hero 区。
- 范围：路由信息架构、首页模块与样式、国内/海外列表页视觉统一与交互、订阅表单与接口调用、可访问性与埋点。

## 路由与信息架构
- 首页：`/` → 新首页（不展示岗位卡片）。
- 列表页：`/jobs?region=domestic|overseas` → 现有岗位列表页（保留搜索/筛选与卡片）。
- 行为：
  - 点击“我在国内” → 跳转 `"/jobs?region=domestic"`。
  - 点击“我在海外” → 跳转 `"/jobs?region=overseas"`。
- 导航：Header 不再放置地域 Tabs；Tabs 仅在列表页顶部居中显示。

## 首页（新）
- 结构：
  - 顶部品牌+一句话价值主张（简洁文案，浅色背景）。
  - 两个大号方形按钮卡片（栅格 2 列，移动端 1 列）：
    - 按钮 A 标题 `我在国内`；副标题 `国内可投的全球远程岗位`；次级说明 `覆盖 China/APAC/UTC+8 及 Global`；点击跳转。
    - 按钮 B 标题 `我在海外`；副标题 `更适合海外华人的全球远程岗位`；次级说明 `覆盖 USA/EU/EMEA/Asia-Pacific 等国家/地区及 Global`；点击跳转。
  - 订阅区块：并列的订阅表单（邮件 + 飞书），置于按钮区块下。
- 视觉：
  - 浅色背景（如 `bg-gradient-to-br from-gray-50 to-gray-100`），统一主色 `#3182CE`，圆角与阴影风格与列表页一致。
  - 大按钮含图标/插画占位，保持风格轻量；悬浮态与点击态一致性（阴影与边框颜色变化）。
- 交互：
  - 按钮支持键盘可达性（`role="button"` + `Enter/Space` 激活）。
  - 订阅区校验与提示统一（成功/失败 Toast）。

## 订阅（邮件 + 飞书）
- 邮件订阅：复用现有 `JobAlertSubscribe` 组件（若已存在），在首页直接渲染，字段包含 `email`；成功后提示并清空。
- 飞书订阅：在同一订阅区提供 `Feishu ID / 飞书号` 输入；
  - 实现方案 A：扩展 `JobAlertSubscribe` 支持第二输入字段并在后端统一保存；
  - 实现方案 B：新增轻量 `FeishuSubscribe` 组件，调用后端聚合接口（例如 `/api/user-profile?action=subscribe_alerts`），后端写 KV/Redis 并（可选）调用飞书机器人 Webhook；
- 防刷与安全：基础频率限制（前端最少 60s 内重复提交禁止）、后端鉴权可选（登录用户优先）。

## 岗位列表页（国内/海外）
- 视觉统一：
  - 移除 Hero 区与插画，顶部保留居中 Tabs（已实现位置），页面背景与按钮风格与首页一致。
  - 保留搜索框、筛选侧栏与卡片列表；整体容器宽度与间距统一（`max-w-7xl`）。
- 行为与逻辑：
  - Tabs 与路由保持一致：切换 `region` 写入 URL，并触发过滤。
  - 分类命中逻辑沿用现有实现（`domestic/global` → 国内；`overseas/global` → 海外；Global 不以仅 `remote` 为准）。

## 前端实现要点
- 页面与组件：
  - 新首页：重构 `src/pages/HomePage.tsx` 为“入口+订阅”的轻量页面；
  - 列表页：更新 `src/pages/JobsPage.tsx`，移除头图/大标题 Hero 区，仅保留搜索与列表；Tabs 继续居中；
  - 订阅组件：复用/扩展 `JobAlertSubscribe`；必要时新增 `FeishuSubscribe`，样式保持一致；
  - Header：确认不再显示地域 Tabs，仅品牌/基础导航。
- 样式：Tailwind 统一色板与阴影，圆角、边框、间距与首页按钮保持一致；移动端断点下栅格切换。
- 可访问性：按钮 `aria-pressed` 与状态提示，表单 `aria-label`、键盘导航与错误提示。

## 后端与接口
- 列表页与收藏链路沿用现有接口（`/api/user-profile`、`/api/data/processed-jobs`）。
- 订阅接口：
  - 优先复用现有订阅存储逻辑；若无则后端合并到 `user-profile` 聚合路由，新增 `subscribe_alerts` 动作（POST，保存 `email`/`feishuId`）。
  - 飞书机器人 Webhook（可选）：通过环境变量配置；失败时容错不影响保存。
- Serverless 数量控制：保持在 ≤ 12，不新增独立函数文件；尽量合并到聚合路由。

## 埋点与监控
- 首页按钮点击：记录 `home_enter_domestic` 与 `home_enter_overseas`；
- 订阅提交：记录 `subscribe_email_success/fail` 与 `subscribe_feishu_success/fail`；
- 列表页 Tabs 点击与跳转：记录 `jobs_tab_domestic/overseas`。

## 验收标准（DoD）
- 首页不展示岗位卡片；按钮与订阅区视觉统一、交互顺畅；邮件与飞书订阅均可提交且有提示。
- 列表页无 Hero 区，背景浅色、Tabs 居中一致；国内/海外分类命中准确；卡片与筛选交互保持现有功能。
- 路由跳转正确，`region` 参数随切换更新；收藏、详情弹窗不受影响。
- 构建通过（`npm run type-check`、`npm run build`）、Serverless 函数数量不超限；部署到预发与生产正常。

## 发布与回滚
- 预发：合并到 `develop` 自动部署；生产：合并到 `main` 自动部署。
- 回滚：保留上一个生产版本，问题时快速回滚；必要时关闭首页订阅入口，列表页功能不受影响。
