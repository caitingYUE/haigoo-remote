# 过往推荐功能测试指南

## 测试步骤

### 1. 基础功能测试
1. 打开浏览器访问 http://localhost:3000
2. 确认页面正常加载，显示今日推荐职位
3. 查看是否有"生成测试历史数据"按钮（开发环境）

### 2. 测试数据生成
1. 点击"生成测试历史数据"按钮
2. 检查控制台是否输出"测试数据已生成！"
3. 确认页面下方出现过往推荐区域

### 3. 渐进式展开测试
1. **初始状态**: 应该看到"查看昨天推荐"按钮
2. **第一次点击**: 
   - 按钮变为"查看更多历史"
   - 显示昨天的推荐职位（3个）
3. **第二次点击**:
   - 按钮变为"查看全部历史"
   - 显示昨天和前天的推荐职位
4. **第三次点击**:
   - 按钮变为"收起"
   - 显示昨天、前天、大前天的推荐职位
5. **第四次点击**:
   - 按钮变回"查看昨天推荐"
   - 隐藏所有过往推荐

### 4. UI/UX 检查点
- [ ] 过往推荐区域有清晰的分隔线
- [ ] 每个日期有明确的标题（昨天推荐、前天推荐、大前天推荐）
- [ ] 职位卡片样式与今日推荐保持一致
- [ ] 按钮状态变化流畅，文字准确
- [ ] 加载状态有适当的loading指示器
- [ ] 响应式设计在不同屏幕尺寸下正常工作

### 5. 数据验证
- [ ] localStorage中存储了正确的历史数据
- [ ] 数据限制在3天内
- [ ] 每天最多显示3个职位
- [ ] 职位数据结构完整（标题、公司、位置等）

## 预期结果
- 所有交互流程顺畅无卡顿
- 数据显示准确完整
- UI界面美观一致
- 用户体验良好
